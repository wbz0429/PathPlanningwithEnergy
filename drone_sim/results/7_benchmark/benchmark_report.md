# Path Planning Benchmark Report

## 实验设置

- **地图**: AirSim Blocks 场景，已知障碍物布局手动构建
- **算法对比**: A* (全局最优 baseline) vs RRT* (4种权重配置)
- **RRT* 每种配置运行 10 次**，取均值和标准差
- **安全边距**: 1.0m
- **RRT* 参数**: 3000 iterations, step_size=1.5m, search_radius=4.0m
- **飞行速度**: 2.0 m/s

## 分析

### 1. RRT* vs A* 最优性差距

- **开阔场景 (Scenario B)**: RRT* Balanced 仅比 A* 多 0.9%，接近最优
- **窄通道场景 (Scenario A)**: RRT* 比 A* 多 ~31%，因为随机采样难以命中 Row2/Row3 的 20m 缝隙，倾向绕行
- **复杂场景 (Scenario C)**: RRT* 比 A* 多 15-25%，方差大（±17-19m），路径质量不稳定

### 2. 消融实验结论

- **Balanced 配置在所有场景均为最优或接近最优**
- Distance-only 和 Energy-only 差异不大（水平飞行时能耗主要由距离决定）
- 有高度变化的场景下能量权重效果会更显著
- Time-priority 表现最差，因为时间优化在等速飞行下等价于距离优化

### 3. RRT* 随机性分析

- Scenario B 方差最小（±0.5m），路径空间简单
- Scenario C 方差最大（±17-19m），存在多条可行路径（绕上/绕下）
- 10 次运行的成功率均为 100%

### 4. 已知局限

- RRT* 在窄通道（narrow passage）场景下表现差，这是采样式规划器的经典弱点
- 当前评测为已知地图上的离线规划，实际飞行中还有感知延迟和建图误差
- 所有场景为同一高度（Z=-3），未测试爬升/下降场景的能量差异
