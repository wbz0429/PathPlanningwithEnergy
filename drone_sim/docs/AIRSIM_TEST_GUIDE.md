### 测试 2：滚动规划飞行测试

```bash
python fly_planned_path.py
```

**测试场景：**
- 起飞到 3m 高度
- 目标设置在障碍物背后（前方 15m，右侧 8m）
- 系统应该：
  1. 持续更新地图
  2. 发现障碍物
  3. 规划绕行路径
  4. 成功到达目标

**观察要点：**

1. **控制台输出：**
   - 每次迭代显示：
     - 当前位置
     - 距离目标的距离
     - 地图更新统计（新增占据体素）
     - 局部目标位置
     - 规划的路径点数

     - 性能统计（感知、建图、规划、执行时间）

2. **可视化窗口：**
   - 3D 视图：红色点=障碍物，蓝色点=当前位置，橙色星=局部目标，绿色星=全局目标
   - 蓝色虚线：当前规划路径
   - 绿色实线：已执行轨迹
   - XY 平面：俯视图
   - XZ 平面：侧视图

3. **飞行行为：**
   - 无人机应该平滑飞行
   - 遇到障碍物时应该绕行
   - 不应该碰撞或急停
   - 最终到达目标附近（1m 以内）

**预期性能：**
- 感知：< 50ms
- 建图：< 50ms
- 规划：< 100ms
- 总周期：< 200ms

**成功标准：**
- ✓ 无碰撞
- ✓ 到达目标（距离 < 1m）
- ✓ 周期时间 < 500ms
- ✓ 生成可视化结果图

## 故障排除

### 问题 1：无法连接 AirSim

**症状：** `Failed to connect to AirSim`

**解决方案：**
1. 确认 AirSim 已启动
2. 检查防火墙设置
3. 重启 AirSim

### 问题 2：规划失败

**症状：** `Path planning FAILED!`

**可能原因：**
1. 目标点在障碍物内部
2. 安全边距设置过大
3. RRT* 迭代次数不足

**解决方案：**
1. 调整目标位置
2. 减小 `safety_margin`（当前 1.0m）
3. 增加 `max_iterations`（当前 2000）

### 问题 3：性能过慢

**症状：** 周期时间 > 500ms

**可能原因：**
1. 地图过大
2. 点云过密
3. RRT* 迭代过多

**解决方案：**
1. 减小 `grid_size`
2. 增加深度图下采样倍数
3. 减少 `max_iterations`

### 问题 4：可视化窗口无响应

**症状：** 窗口卡住或不更新

**解决方案：**
1. 关闭可视化：设置 `visualize=False`
2. 减少可视化频率
3. 降低显示的点云数量

```python
])
```

### 变体 2：不同规划参数

调整滚动规划配置：

```python
receding_config = {
    'local_horizon': 8.0,      # 增加局部视野
    'execution_ratio': 0.3,    # 减少执行比例（更频繁重规划）
    'flight_velocity': 3.0,    # 提高飞行速度
}
```

### 变体 3：禁用可视化（提高性能）

```python

    'visualize': False  # 关闭实时可视化
}
```
