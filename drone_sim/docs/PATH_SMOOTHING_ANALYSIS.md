# EA-RHP 路径平滑问题分析与改进方案

## 问题描述

### 现象
滚动规划（Receding Horizon）输出的飞行轨迹在拐弯处存在明显的折角，路径呈现"折线段拼接"的特征，不够平滑。

### 导师提问
> "如果结果是一段路径，并且这个路径是迭代更新的话，为啥会出现类似于几段拼接有拐角的情况呢？"

## 原因分析

拐角来自两个层面：

### 1. RRT* 采样特性（次要原因）
- RRT* 通过随机采样点构建路径树，最终路径是采样点之间的折线段连接
- 有限迭代次数（3000次）下，路径不会完全光滑
- 但在单次规划中，B-spline 后处理可以有效解决（转角从 42° 降到 3.8°）

### 2. 滚动规划的拼接（主要原因）
- 每次规划周期只规划 8m 局部路径，执行 30% 后重新规划
- **每次重规划是独立的**：完全抛弃上次路径，基于新地图从零开始 RRT*
- **没有方向连续性约束**：新路径的起始方向可能和上次结束方向不一致
- **局部路径太短**：8m horizon 内 RRT* 通常直接找到直线路径（仅2个航点），B-spline 需要至少4个点才能生效

### 示意图

```
迭代1规划方向:  ——→
迭代2规划方向:       ↗    （方向突变，产生拐角）
迭代3规划方向:            →
```

两次规划之间没有"记忆"，拼接处产生方向突变。

## 当前已实现的优化

### B-spline 路径平滑（已完成）

在 `planning/rrt_star.py` 的 `_smooth_path()` 中实现：

1. 适度简化：每次最多跳 10m（保留足够中间点）
2. B-spline 三次样条拟合（`scipy.interpolate.splprep`）
3. 碰撞安全检查：平滑后路径如果碰撞则回退

**效果（全局规划 70m 路径）：**

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 路径点数 | 3 | 271 |
| 最大转角 | 41.9° | 3.8° |
| 平均转角 | 41.9° | 0.7° |
| >30°拐角数 | 1 | 0 |
| 路径长度 | 77.8m | 82.1m |

**局限：** 滚动规划中每次局部路径仅2个点，B-spline 触发不了。

### 动态路径安全检查（已有）

在 `planning/receding_horizon.py` 执行循环中：
- 执行前碰撞检测：检查当前位置安全距离
- 路径安全检查：检查到下一航点的路径是否无碰撞
- 执行后安全检查：到达航点后更新地图，检查前方安全性
- 发现危险立即中止执行，触发重规划

## 待实现的改进方案

### 方案1: 路径修复（Path Repair）— 推荐优先实现

**思路：** 不是每次从零开始规划，而是复用上次路径

```
if 上次路径仍然安全:
    继续执行上次路径
elif 上次路径部分被堵:
    只修复被堵的部分
else:
    完全重新规划
```

**优点：**
- 保证路径连续性（不会有拼接拐角）
- 减少不必要的重规划（节省计算）
- 实现相对简单

**实现要点：**
- 每次循环先检查剩余路径是否安全（用 ESDF）
- 安全则直接继续执行
- 不安全则从被堵点开始局部重规划，拼接到原路径后半段

### 方案2: 方向连续性约束

**思路：** 规划新路径时，约束起始方向和上次飞行方向一致

```python
# 记录上次飞行方向
last_direction = trajectory[-1] - trajectory[-2]

# 新规划时，在起点附近插入一个"方向引导点"
guide_point = current_pos + normalize(last_direction) * 2.0

# RRT* 规划时强制经过 guide_point
path = rrt.plan(current_pos, local_goal, via_point=guide_point)
```

**优点：** 直接消除拼接处的方向突变
**缺点：** 可能限制避障灵活性

### 方案3: Warm Start RRT*

**思路：** 用上次的搜索树作为初始树，新一轮在此基础上扩展

**优点：** 路径质量更高，收敛更快
**缺点：** 实现复杂，需要处理地图变化后的树节点失效

### 方案4: 轨迹优化后处理

**思路：** 在滚动规划层面，对已执行的轨迹做全局平滑

```python
# 每次执行完新航点后，对最近 N 个航点做 B-spline 拟合
# 用平滑后的轨迹替换原始航点
recent_trajectory = executed_trajectory[-N:]
smoothed = bspline_smooth(recent_trajectory)
```

**优点：** 不影响规划逻辑，纯后处理
**缺点：** 无人机已经飞过了，只能改善后续航点
